# خطة تحسين تقارير المرضى في صفحة التقارير

## نظرة عامة
تهدف هذه الخطة إلى تحسين أداء وعرض تقارير المرضى في صفحة التقارير الرئيسية. تشمل التحسينات تحسين الأداء، تحسين التصميم، وإضافة ميزات جديدة لتوفير تجربة مستخدم أفضل.

## التحليل الحالي

### المكونات الحالية لتقارير المرضى
- **بطاقة KPI**: إجمالي المرضى
- **رسم بياني**: توزيع المرضى حسب الفئات العمرية (مخطط دائري)
- **البيانات**: إجمالي المرضى، المرضى الجدد، الفئات العمرية

### نقاط الضعف
1. **الأداء**:
   - تحميل البيانات غير محسن (استدعاء مزدوج لنفس الخدمة)
   - عدم استخدام memoization للرسوم البيانية
   - عدم دعم التحميل التدريجي

2. **التصميم**:
   - عرض بطاقة واحدة فقط للمرضى (إجمالي المرضى)
   - رسم بياني واحد فقط (توزيع العمر)
   - عدم استخدام نطاق التاريخ في جلب البيانات
   - تصميم بسيط بدون تفاعل كافي

3. **الميزات الناقصة**:
   - عدم عرض المرضى الجدد بشكل واضح
   - عدم وجود اتجاهات زمنية
   - عدم وجود تصدير أو طباعة
   - عدم وجود جدول تفصيلي

## التحسينات المقترحة

### 1. تحسينات الأداء
- إصلاح استدعاء خدمة البيانات (استدعاء واحد بدلاً من اثنين)
- تطبيق React.memo على مكونات الرسوم البيانية
- استخدام useMemo لحساب بيانات الرسوم البيانية
- تحسين ResponsiveContainer لتجنب إعادة الحساب

### 2. تحسينات التصميم
- إضافة بطاقة KPI للمرضى الجدد
- تحسين توزيع البطاقات والألوان
- إضافة رسوم بيانية إضافية (خط زمني للمرضى الجدد)
- تحسين ألوان وتفاعل الرسوم البيانية
- إضافة أيقونات وتحسين التباعد

### 3. ميزات جديدة
- ربط نطاق التاريخ بجلب البيانات
- إضافة جدول تفصيلي للفئات العمرية
- إضافة خيارات تصدير (PDF، Excel)
- إضافة فلاتر إضافية (نوع المريض، الجنس، إلخ)

## خطة التنفيذ

### المرحلة 1: إصلاحات الأداء الأساسية
1. إصلاح استدعاء خدمة البيانات في ReportsPage.tsx

3. إضافة رسم بياني خطي للمرضى الجدد عبر الزمن
4. تحسين تفاعل الرسوم البيانية

### المرحلة 3: ميزات متقدمة

4. إضافة فلاتر إضافية

## المتطلبات التقنية

- أدوات اختبار الأداء

- ضمان إمكانية الوصول والاستجابة على الأجهزة المحمولة
