# Plan for New Reports Section

## Overview
This plan outlines the design and implementation of a new reports section in the CuraSoft application. The section will provide detailed and printable summaries for various aspects of the clinic's operations, including financial data, patient information, doctor performance, treatments, and supplier details.

## Requirements
The new reports section should include:
1. **Financial Reports**:
   - Revenue and payment details
   - Expense breakdowns
   - Doctor payment ratios
   - Daily expenses
   - Daily financial details (revenue, expenses, and net profit per day)
   - Monthly detailed reports

2. **Patient Reports**:
   - Detailed patient information
   - Treatment histories
   - Financial summaries

3. **Doctor Reports**:
   - Performance metrics
   - Treatment statistics
   - Financial contributions

4. **Treatment Reports**:
   - Treatment frequency and types
   - Revenue generated by treatment type

5. **Supplier Reports**:
   - Material suppliers
   - Lab suppliers
   - Invoice and payment histories

## Data Sources
The following data sources and interfaces will be utilized:
- `Payment` interface for revenue and payment data
- `Expense` interface for expense tracking
- `TreatmentRecord` interface for treatment details
- `DoctorPayment` interface for doctor compensation
- `SupplierInvoice` interface for supplier financials
- `Patient` interface for patient information
- `Dentist` interface for doctor details
- `Supplier` interface for supplier information
- `TreatmentDefinition` interface for treatment types

## Design Layout

### 1. Reports Dashboard
A central dashboard that provides quick access to all report types with summary cards and navigation options.

### 2. Report Categories
- **Financial Reports**: Revenue, expenses, doctor payments, and supplier invoices
- **Patient Reports**: Patient details, treatment histories, and financial summaries
- **Doctor Reports**: Performance metrics, treatment statistics, and financial contributions
- **Treatment Reports**: Treatment frequency, types, and revenue
- **Supplier Reports**: Material and lab suppliers, invoice histories

### 3. Printable Layouts
Each report will have a printable version with:
- Clinic header and contact information
- Date range filters
- Summary statistics
- Detailed tables with relevant data
- Professional formatting for printing

## Implementation Steps

### Step 1: Create Reports Dashboard Component
- Design a dashboard layout with navigation to different report types
- Include summary cards for quick insights
- Implement date range filters for all reports

### Step 2: Develop Financial Reports
- **Revenue Report**: Display all payments and treatment records
- **Expense Report**: Show all expenses and doctor payments
- **Daily Financial Report**: Show daily revenue, expenses, and net profit
- **Supplier Report**: List all supplier invoices and payments
- **Monthly Summary**: Provide a comprehensive monthly financial overview

### Step 3: Develop Patient Reports
- **Patient Details Report**: Include patient information, medical history, and contact details
- **Treatment History Report**: List all treatments performed on a patient
- **Financial Summary Report**: Show payments and outstanding balances for patients

### Step 4: Develop Doctor Reports
- **Performance Report**: Display treatment statistics and patient counts
- **Financial Report**: Show earnings and payment histories
- **Treatment Summary**: List all treatments performed by the doctor

### Step 5: Develop Treatment Reports
- **Treatment Frequency Report**: Show the most common treatments
- **Revenue by Treatment Report**: Display revenue generated by each treatment type

### Step 6: Develop Supplier Reports
- **Material Supplier Report**: List all material suppliers and their invoices
- **Lab Supplier Report**: Show all lab suppliers and their cases
- **Invoice History Report**: Display invoice and payment histories for suppliers

### Step 7: Implement Printable Layouts
- Create a reusable printable layout component
- Ensure all reports can be rendered in a print-friendly format
- Include clinic branding and professional styling

### Step 8: Integrate with Existing Components
- Connect the new reports section to the existing navigation
- Ensure data consistency with the financial and patient management sections
- Implement proper error handling and loading states

## Technical Specifications

### Components to Create
1. **ReportsDashboard.tsx**: Main dashboard for accessing all reports
2. **FinancialReports.tsx**: Component for financial reports
3. **PatientReports.tsx**: Component for patient reports
4. **DoctorReports.tsx**: Component for doctor reports
5. **TreatmentReports.tsx**: Component for treatment reports
6. **SupplierReports.tsx**: Component for supplier reports
7. **PrintableReportLayout.tsx**: Reusable layout for printable reports

### Data Flow
- Use the existing `useClinicData` hook to fetch clinic data
- Implement filters for date ranges, doctors, suppliers, and treatment types
- Ensure all reports are responsive and printable

### Styling
- Use Tailwind CSS for styling
- Ensure consistency with the existing design system
- Optimize layouts for both screen and print media
